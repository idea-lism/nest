# PEG code generating

src/peg_gen.c generates LLVM IR and C header code from analyzed PEG scope closures.

# Input

```c
typedef struct {
  ScopeClosure* scope_closures; // darray, fully analyzed (from peg_analyze)
  Symtab tokens;                // forwarded from PegAnalyzeInput
  Symtab scope_names;           // forwarded from PegAnalyzeInput
  Symtab rule_names;            // forwarded from PegAnalyzeInput
  int memoize_mode;
  const char* prefix;
  int verbose;
} PegGenInput;
```

create function `void peg_gen(PegGenInput* input, HeaderWriter* hw, IrWriter* w)`:

See [peg_analyze](peg_analyze.md) for how `PegGenInput` is produced.

# Code gen
## Per-scope context

The IR generation uses a `PegIrScopeCtx` to hold per-scope constants, avoiding repeated computation:

```c
typedef struct {
  IrVal peg_table;
  int64_t sizeof_col;
  int64_t bits_bucket_size;
  int64_t slot_byte_offset; // precomputed: bits_bucket_size * 8 + slot_index * 4
  int64_t tag_byte_offset;  // precomputed: tag_bit_index * 8
  int32_t tag_bit_count;    // forwarded from ScopedRule, for multi-bucket tag writeback
} PegIrScopeCtx;
```

## Memoize strategy dispatch

Memoize read/write/deny are split into per-mode strategy functions, dispatched via function pointer tables:

- `_memoize_read_{none,naive,shared}(ctx, sc, sr, labels)`: emit memoize lookup → branches to material_parse, fail_bb, or fast_ret→done_bb
- `_memoize_write_{none,naive,shared}(ctx, sc, sr, start_col)`: emit cache-on-success after material parse
- `_memoize_deny_{none,naive,shared}(ctx, sc, sr, fail_col)`: emit cache-on-fail in parse_fail block

The main per-rule loop dispatches once per scope:
```c
MemoizeReadFn memoize_read = _memoize_read_fns[memoize_mode];
// ... per rule:
memoize_read(&ctx, &sc, sr, &ml);
// ... material parse ...
memoize_write(&ctx, &sc, sr, start_col);
// ... parse fail ...
memoize_deny(&ctx, &sc, sr, fail_col);
```

## GEP and tag helpers

Slot/tag pointer computation and tag writeback use `@gep_slot`, `@gep_tag`, `@tag_writeback` IR helpers (see [peg_ir](peg_ir.md)). These replace inline GEP offset math.
`define internal parse_{scope_name}(TokenTree* tt, stack_ptr)`: callable from the IR generated by [VPA](vpa.md) because they live in the same IRWriter.

In the funciton, we allocate & init table for the scope, then lower to fine-graned parsing procedures:
```pseudo
%tc = tt_current(tt)
%col = i64 0
%peg_table = tt_alloc_memoize_table(%tc, sizeof_col)
%tc->value = %peg_table
%tokens = tc->tokens
{ peg_ir_emit_call(ctx, entrance_rule_name) } ; pushes stack with current col

{scope_name}$foo:
  { reads memoize table, goto fail_bb / fast_ret / material_parse }

fast_ret:
  %parsed_tokens = { memoized size }
  %col += %parsed_tokens
  br done_bb

material_parse:
  { if rule has tag bits }
    %tag_bits = 0; { if tag_bit_count > 64: one %tag_bits_N per bucket }
  { end }

  { peg_ir_emit_parse(ctx, scoped_rule.body, parse_fail) }

  %start_col = @top()
  { if rule has tag bits }
    %bits = gep(%peg_table[%start].bits);
    { if tag_bit_count <= 64 }
      %bits[{tag_bit_index}] &= ~{tag_clear_mask}
      %bits[{tag_bit_index}] |= %tag_bits
    { else }
      for w in 0..ceil(tag_bit_count/64):
        %bits[{tag_bit_index + w}] = %tag_bits_{w}  ; dedicated buckets, full overwrite
    { end }
  { end }
  { cache in memoize table }

  %parsed_tokens = %col - %start_col
  br done_bb

parse_fail:
  %col = @top()
  { denies memoize table }
  br fail_bb

fail_bb:
  %parsed_tokens = -1
  br done_bb
done_bb:
  { peg_ir_emit_ret(ctx) } ; pops stack

; a multiplier rule
{scope_name}$foo$1:
  ... same as defined rule

...

ret %parse_result
```

To access memoize table:

- Read memoize table slot
  - `%peg_table[%col].slots[%slot_index]`
- Write memoize table slot
  - `%peg_table[%col].slots[%slot_index] = val`

When memoize_mode=`shared` mode, we also have:

- `define internal @bit_test(ptr table, i64 col, i64 col_size, i64 seg_offset, i64 rule_bit)`: Test rule's bit in the segment. Returns `i1` (1 = may match, 0 = proven fail).
  - `seg_offset = segment_index * sizeof(i64)`
- `define internal @bit_deny(ptr table, i64 col, i64 col_size, i64 seg_offset, i64 rule_bit)`: Clear rule's bit (cache failure).
- `define internal @bit_exclude(ptr table, i64 col, i64 col_size, i64 seg_offset, i64 segment_mask, i64 rule_bit)`: Keep only this rule's bit, zero out others in segment (cache exclusive match).

In the end the memoize table is associated to each `TokenChunk`.

# Stack safety

`parse_{scope_name}` should be stack-protected with canary `sspreq`.

For native call stack: when alloca is dynamic (not single-time funciton-top), add LLVM hint on alloca lifetime for this alloca:
- `@llvm.lifetime.start`
- `@llvm.lifetime.end`

# User API: parse tree retrieval

When parse succeeds, VPA will return a PegRef for `main` rule.

Node struct fields and loader code are generated from precomputed `ScopedRule.node_fields` (see [peg_analyze](peg_analyze.md)). Each `NodeField` describes one struct field with its name, type (PegRef vs PegLink), and per-scope slot row assignments for the loader's cursor advancement.

```c
typedef struct {
  TokenChunk* tc;
  int64_t col;
  int64_t row; // absolute i32 row in { i64 bits[]; i32 slots[] }
} PegRef;

// for multiplier/interlaced rules, similar to
typedef struct {
  TokenChunk* tc;
  int64_t col;
  int64_t col_size_in_i32;

  int64_t lhs_bit_index;
  int64_t lhs_bit_mask;
  int64_t lhs_row; // >= 0: call lhs (read slot), -1: no link, -2: term lhs (advance by 1, check lhs_term_id)

  int64_t rhs_bit_index;
  int64_t rhs_bit_mask;
  int64_t rhs_row; // >= 0: call separator (read slot), -1: no separator, -2: term separator (advance by 1)
} PegLink;

typedef Node_{rule_name} {
  struct {
    bool {tag1}: 1;
    bool {tag2}: 1;
    ...
    { for each uint64_t bucket needed: ceil(tag_count / 64) }
    uint64_t _padding{w}: {64 - used_bits_in_this_bucket};
    { end }
  } is; // sizeof = ceil(tag_count / 64) * 8
  PegRef {child0};
  PegLink {multiplier_child1};
  PegRef {child1};
  ...
};
```

For repeated name fields, the ocurrences are auto-renamed to `field$1, field$2`.

Tag bits can be assigned all at once, the loading function is like:

```c
{prefix}_load_{normal_rule_name}(PegRef ref) {
  Node_foo $1 = {0};
  int64_t* $table = ref.tc->value;
  switch (ref.tc->scope_id) {
    case {rule_used_in_scopes[0]}: {
      int64_t* $col = $table + ref.col * {col_size_in_i64 of this scope}
      { if tag_bit_count <= 64 }
        ((uint64_t*)&$1.is)[0] = ($col[{tag_bit_index}] >> {tag_bit_offset}) & {(1ULL << tag_bit_count) - 1};
      { else }
        for w in 0..ceil(tag_bit_count/64):
          ((uint64_t*)&$1.is)[w] = $col[{tag_bit_index + w}];  // dedicated buckets, offset=0
      { end }
      break;
    }
    case {rule_used_in_scopes[1]}: {
      int64_t* $col = $table + ref.col * {col_size_in_i64 of this scope}
      ... // we have different number allocations for different scopes
      break;
    }
    ...
  }
  return $1;
}
```

Note if rule name is a scope, we have to expand the child token chunk first, and then the loading is like normal_rule_name.
- Note that we need token_tree for the expansion, so the chunk was created by tt_push_assoc in [VPA](vpa.md)
- With `aux_value` as token_tree we can expand

Generic link iterating functions should be defined as:

```c
// NOTE: for `term` lhs / naive mode that's much simpler & trivial,
//       but the `call` lhs is more difficult so we only list what will happen for `call` case here

bool {prefix}_has_elem(PegLink* l) {
  if (l->col >= darray_size(l->tc->tokens)) {
    return false;
  }
  int32_t* $col = (int32_t*)l->tc->value + l->col_size_in_i32 * l->col;
  uint64_t* $bits = (uint64_t*)$col;
  return ($bits[l->lhs_bit_index] & l->lhs_bit_mask) && $col[l->lhs_row] >= 0;
}

void {prefix}_get_next(PegLink* l) {
  int32_t lhs_size = (l->lhs_row < 0) ? 1 : $col[l->lhs_row];
  l->col += lhs_size;
  if (l->rhs_row == -2) l->col += 1;          // term separator
  else if (l->rhs_row >= 0) l->col += $col[l->rhs_row]; // call separator
}

PegRef {prefix}_get_lhs(PegLink* l) {
  return (PegRef){l->tc, l->col, l->lhs_row}
}

PegRef {prefix}_get_rhs(PegLink* l) {
  return (PegRef){l->tc, l->col + lhs_size, l->rhs_row};
}
```

A typical using pattern is:

```c
PegRef ref = ...;
Node_foo foo = {prefix}_load_foo(ref);
// foo = [
//   bar+ : tag1
//   baz bar : tag2
// ]
if (foo.is.tag1) {
  for (PegLink l = foo.bar; {prefix}_has_elem(&l); {prefix}_get_next(&l)) {
    Node_bar bar = {prefix}_load_bar({prefix}_get_lhs(&l));
    ...
  }
} else if (foo.is.tag2) {
  Node_baz baz = {prefix}_load_baz(foo.baz);
  Node_bar bar = {prefix}_load_bar(foo.bar$1);
}
```

# Node field naming scheme

For example, the rule

```nest
while_stmt = "while" @lparen expr @rparen block
```

will generate a node definition of:

```c
typedef struct {
  PegRef _lit_while; // @lit.while
  PegRef _lparen; // @lparen
  PegRef expr;
  PegRef _rparen; // @rparen
  PegRef block;
} Node_while_stmt;
```

just replace the `@` & `.` with `_` for tokens.
